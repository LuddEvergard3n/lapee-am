# a11y.js — Documentação Técnica

**Localização:** `js/a11y.js`

## Responsabilidade

Controlar as preferências de acessibilidade do usuário e aplicá-las ao DOM.
Registrar atalhos de teclado globais. Oferecer utilitário de foco para navegação por leitores de tela.

## Mecanismo de aplicação

As preferências são aplicadas como classes CSS no elemento `<html>` (`document.documentElement`).
O CSS em `main.css` usa seletores como `html.fonte-grande`, `html.alto-contraste` e `html.tema-escuro` para sobrescrever variáveis CSS.

| Preferência | Classe em `<html>` | Efeito CSS |
|---|---|---|
| Fonte grande | `fonte-grande` | `font-size: 120%` no `html` |
| Alto contraste | `alto-contraste` | Sobrescreve paleta com cores de alto contraste |
| Tema escuro | `tema-escuro` | Sobrescreve paleta com cores escuras |

As combinações são suportadas: `html.alto-contraste.tema-escuro` tem seletor próprio no CSS.

## API pública

### `applyAll()`

Lê as preferências do store e aplica todas as classes correspondentes no `<html>`.
Deve ser chamado:
1. Na inicialização (`app.js`), antes de qualquer renderização.
2. Após qualquer mudança de preferência (as funções toggle já fazem isso internamente).

### `toggleFonteGrande()`
### `toggleAltoContraste()`
### `toggleTemaEscuro()`

Invertem a preferência correspondente via `setState` do store e chamam `applyAll()`.
A mudança é persistida automaticamente porque o store serializa `prefs` no localStorage.

### `registerKeyboardShortcuts()`

Registra um único listener `keydown` em `document`. Atalhos disponíveis:

| Tecla | Ação |
|---|---|
| `Alt + F` | `toggleFonteGrande()` |
| `Alt + C` | `toggleAltoContraste()` |
| `Alt + D` | `toggleTemaEscuro()` |
| `Escape` | `_closeOpenDrawers()` |

`e.preventDefault()` é chamado nos atalhos Alt+tecla para evitar conflitos com o navegador.

### `focusById(id)`

Move o foco programaticamente para o elemento com o ID fornecido.
Adiciona `tabindex="-1"` ao elemento se necessário para permitir foco via script.
Útil após navegação entre páginas para reposicionar o foco para leitores de tela.

**Nota:** esta função está disponível mas não é chamada automaticamente nesta versão. Pode ser usada em extensões futuras.

## Comportamento do Escape

`_closeOpenDrawers()` verifica se o elemento `#sidebar` tem a classe `open`.
Se sim, remove a classe e atualiza o `aria-expanded` do botão `#btn-menu`.

## Considerações de acessibilidade no CSS

- `aria-live="polite"` nos elementos de feedback garante anúncios sem interromper o leitor.
- `aria-label` e `aria-labelledby` são usados em todas as regiões e controles interativos em `ui.js`.
- `role="radiogroup"`, `role="radio"`, `aria-checked` são aplicados nos botões de opção de `marcar.js`.
- `role="progressbar"` com `aria-valuenow/min/max` na barra de progresso da unidade.
- Contraste mínimo: o tema padrão usa cores que atendem WCAG AA na maioria dos pares texto/fundo. O modo alto contraste visa WCAG AAA.
